That single permission gate — silent for routine tasks, explicit for sensitive data — is a new interaction pattern: the agent classifies task sensitivity and escalates to the user only when the data-locality decision actually matters.

---

The macro signal across all six products this week is that **execution architecture is becoming a user-facing design problem**. When Claude tells you 47/100 subagents are done, when Grok Build lets the model kill its own monitors, when Perplexity's Personal Computer silently routes a medical record query to on-device processing and asks permission before it doesn't, when Excel flags which cells an agent touched — these are all the same design move: collapsing the gap between what the system is *doing* and what the user can *see and control*. The industry is not yet converged on vocabulary or pattern: "monitors," "sessions," "workflows," "Computer," "Show Changes" are all pointing at the same underlying need. But the need itself — legible, stoppable, attributable execution — is now the dominant design problem across every product in this landscape simultaneously.
