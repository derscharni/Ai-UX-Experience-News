**April 14, 2026**

Good morning. Today's briefing highlights a shift toward deep user customization and strategic interface refinement. While some agents are expanding their capabilities to allow users to build custom workflows and compare models, others are actively retreating from the UI to reduce visual noise.

## Product Highlights

### Claude's Skill Creator

Anthropic has taken a major step in empowering power users with the introduction of a built-in "Skill Creator" in Claude. [1] This feature moves beyond simple prompt templates. Users can now define custom skills with structured fields (task, goal, steps, output format) and trigger them instantly via a slash command. This represents a significant evolution in interaction design, allowing users to build their own micro-applications and standardized workflows directly within the chat interface, rather than relying solely on Claude's default behaviors.

### Perplexity's Model Council

Perplexity continues to innovate on the research UX with the launch of "Model Council." [2] Instead of forcing the user to trust a single model's output, this feature allows users to compare answers from various frontier models (such as OpenAI's GPT series and DeepSeek's R1) side-by-side. This is a powerful trust-building pattern. It acknowledges that no single model is perfect for every query and positions the user as an editor synthesizing multiple perspectives, rather than a passive recipient of a single AI-generated truth.

### Microsoft's "Noise Reduction"

The most discussed UX topic this week remains Microsoft's strategic rollback of Copilot UI elements. [3] The company is actively removing Copilot branding and "unnecessary" buttons from core Windows 11 applications like Notepad and the Snipping Tool. Interestingly, the underlying AI functionalities remain accessible via menus or shortcuts; it is purely the *visible presence* of the AI that is being reduced. This confirms a broader industry trend: the initial phase of slapping an "AI sparkle" button on every UI surface is ending, replaced by a desire for cleaner, less intrusive interfaces where AI operates quietly in the background.

## Broader Trends: The Developer UX

We are seeing a renewed focus on the developer and power-user experience. Google's rollout of command-line interaction for Gemini, alongside a new UX toolkit, provides developers with the tools to build custom, pixel-based applications powered by Gemini. [7] Similarly, the rumored ChatGPT "superapp" with a dedicated Scratchpad UI for parallel Codex tasks suggests that the standard chat interface is no longer sufficient for complex coding workflows. [6] The UX of AI is branching: simpler, quieter interfaces for general consumers, and dense, highly customizable dashboards for creators and developers.
