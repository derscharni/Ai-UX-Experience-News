**March 9, 2026**

A fresh, concise overview of the latest user experience developments in the AI agent landscape. From hackathons dedicated to “Agentic UI” to a major redesign of ChatGPT’s core experience, the industry is starting to ask: what should an agent actually *look* and *feel* like?

## Product Highlights

### ChatGPT Breaks Out of the Box: The Canvas Is Here

In its most significant interface overhaul to date, OpenAI has unveiled a major redesign for ChatGPT centered on a new **“Canvas” interface** [1]. This moves beyond the traditional linear, chronological chat feed to a freeform, document-style workspace. The goal is to make it easier to revise and structure larger writing and coding projects. It’s a fundamental shift in the core interaction pattern, acknowledging that complex tasks require more than a simple back-and-forth conversation. The agent is no longer just a conversationalist; it’s a collaborative partner in a persistent workspace.

### Manus and the Search for “Agentic UI”

While OpenAI redesigns, the community is actively trying to define the design language for this new era. The **Manus Academy’s “AI Agent Glow Up Hackathon”** is a perfect example, with a stated goal of developing “Agentic UI” [2]. The event’s judging criteria explicitly included “Aesthetics: Usability and interface quality,” with a call for participants to build agents that “look as good as they think.” The hackathon encouraged the use of multiple models (Gemini for visual reasoning, Claude for architecture) and the ability to switch “brains” mid-conversation — a new interaction pattern that treats the underlying LLM as a swappable component.

### Microsoft’s Agentic SharePoint Admin

Microsoft is applying the agentic model to the complex world of enterprise IT administration. A new **agentic SharePoint Admin Agent** gives administrators a conversational interface for managing their environment [3]. The agent has specific, auditable “skills” for tasks like analyzing site permissions, managing the site lifecycle, and providing storage insights. The focus here is on governance and trust — building an agent that is not just powerful, but also transparent and reliable enough for critical enterprise workflows.

---

## The Bigger Picture: The A2UI Model

The most important new concept to emerge this week is the **“A2UI” (agent-to-user-interface) model** [7]. This describes a system where the agent itself can dynamically render a user interface based on the specific context of a task. Instead of a static, one-size-fits-all chat window, the agent could generate buttons, sliders, forms, or even entire dashboards on the fly to best suit the user’s needs.

This is the next logical step. We’ve seen glimpses of it in Claude’s use of buttons and checkboxes to gather information, and it’s the philosophical underpinning of ChatGPT’s new Canvas. The agent isn’t just living *inside* the UI anymore; the agent is *becoming* the UI.

This marks a maturation of the space. The initial race was about raw intelligence. The current race is about agentic capability. The next race will be about the user experience. The winners will be the platforms that build agents that are not only smart and capable, but also intuitive, elegant, and a genuine pleasure to work with.
