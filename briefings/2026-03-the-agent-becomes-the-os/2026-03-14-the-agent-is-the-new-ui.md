# UX Briefing: The Agent Is the New UI

**March 14, 2026**

A fresh, concise overview of the latest user experience developments in the AI agent landscape. Today’s theme: **The agent is the new UI.** After a week of deep integrations, the focus is now shifting to how agents *present* themselves. From dynamically generated interfaces to interactive in-chat visuals, the static chat window is giving way to a more fluid, context-aware, and generative user experience.

## Product Highlights

### Claude and the Generative UI

The most forward-looking UX concept to emerge this week is Anthropic’s **“Generative UI”** [1]. Instead of just describing an interface, Claude can now *construct* it on the fly, presenting users with the buttons, sliders, and inputs needed for a specific task. This is a fundamental shift from conversational UI to generative UI. Paired with the new ability to create **interactive charts and diagrams** directly in the chat window [2], Claude is pioneering an experience where the agent builds the tools you need, right when you need them.

### Google’s Agent-Native Workspace

Google’s deep integration of Gemini into its Workspace suite continues to evolve the UX of productivity. The latest updates introduce a **new bottom bar and side panel for prompting in Google Docs**, and a powerful **“Fill with Gemini” feature in Sheets** that can auto-populate entire tables with generated or web-searched data [3]. This isn’t just about adding AI features; it’s about redesigning the core interface of these applications to be agent-native. The prompt is no longer a separate window but a first-class citizen of the document itself.

### The Cross-Device Digital Worker

Perplexity’s vision for an always-on “digital worker” takes a major step forward with the rollout of **Perplexity Computer to all iOS users** [8]. This enables true cross-device continuity. A user can start a research task on their phone, and the agent will continue working on the connected Mac mini, with the context passed seamlessly between devices. This is a powerful new interaction pattern that moves the agent from a session-based tool to a persistent, background collaborator.

---

## The Bigger Picture: A Standard for Agentic UI

As agents become more capable of interacting with software, a critical question has emerged: how do we prevent every app from needing a custom-built agent? A new collaboration between **Oracle, CopilotKit, and Google** aims to solve this by creating a set of open standards for agent-UI interaction, including **Open Agent Specification, AG-UI, and A2UI** [11].

The goal is “plug-and-play portability” for agentic applications. This is the plumbing that could enable a future where any agent can interact with any application, just as a human can. It’s a foundational effort that parallels the way HTML and web browsers standardized how we access information. This standardization is also being met from the other side, with platforms like **monday.com** building infrastructure to allow AI agents to sign up, authenticate, and operate within their platform as if they were human users [12].

The era of the standalone chatbot is over. The future is a world of interoperable, autonomous agents, and the industry is now building the roads and railways for them to travel on.
